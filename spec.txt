*Overview*

Develop a command line scraper application, which will scrape a specified job site, and save it's jobs
into an xml file.

*Parameters to be accepted from the command line app*

./scraper_name -u <url> -u <username> -p <password> --page <page default to 1> --post-params <url encoded parameters>  -f <file location>

e.g.

./barrickscraper -h "https://barrick.taleo.net/careersection/2/jobdetail.ftl" -u admin -p admin --page 1 --post-params "&search=engineer&location=New%20York"  -f /home/scrapers/barrick-20140101120000.xml

Notes:

(1) SSL certificate checks/verifications should be disabled, but HTTPS needs to be supported
(2) Text highlighted in blue are optional
(3) Please do not customized the parameters, you can add new features but you should be able to support the format exactly.
(4) for GET, parameters should be encoded directly on the url.
e.g. https://barrick.taleo.net/careersection/2/jobdetail.ftl?search=accountant
(5) Scraper should return / echo the total count of data scraped, if there is a error, pls use the following  error format

error|timeout - for timeouts
error|nojobs - when zero jobs are parsed
error|<error message> - for custom errors.

*Scraper Required Functionality:* (Basic Process Flow)

(1) Scraper should be able send GET / POST Requests
(2) Process Server Responses by capturing tag and setting cookies, for
handling sessions
(3) Able to parse HTML, XML, JSON, or custom format data
(4) Able to initiate paging requests.
(5) Save results in an xml file.

*Data to be parsed*

  * key - unique identifier * - required
  * title - 200 character limit * - required
  * company - 200 character limit * - required
  * city - 200 character limit * - required
  * state - 200 character limit
  * country - 200 character limit
  * url - 200 character limit * - required
  * snippet - 1000 character limit * - required
  * date_posted (yyyy-mm-dd) * - required
  * date_added (yyyy-mm-dd)
  * provider - name of scraper, 200 character limit (to be given by Andrew)

*Format of XML File.*

<?xml version="1.0" ?>
<data scraper="name of scraper" date="datetime parsed" >
    <job>
        <key/>
        <title/>
        <company/>
        <city />
        <state />
        <country />
        <url />
        <snipppet />
        <date_posted />
        <date_added />
        <provider />
    </job>
</data>

Notes:

(1) All data tags should use the CDATA Tag.

http://www.w3schools.com/xml/xml_cdata.asp

e.g. <title><![CDATA[2014 Barrick Internship - Environmental Engineering-072364 ]]></title>

*Filename format*

XML File should be saved as <Scraper Name>-<YYYYMMDDHHIISS>.xml
e.g. barrick-20140101120000.xml

Scraper name should be one word, lowercased, letters and numbers only.
